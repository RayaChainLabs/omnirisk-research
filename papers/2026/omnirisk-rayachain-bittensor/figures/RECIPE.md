# Figure recipe — OMN-152 v0.11 (polished by OMN-160)

## Overview

Five vector PDFs in this directory, all reproducible from `_make_figures_polished.py`.

| File | Status | Description |
|---|---|---|
| `five-engine-architecture.pdf` | polished | Block diagram of five engines + shared substrate |
| `rya-defence-cycle-prices.pdf` | polished | RYA price (µUSD) over defence cycle with buy markers |
| `rya-defence-cycle-liquidity.pdf` | polished | Pool liquidity envelope over defence cycle |
| `five-engine-dependency-graph.pdf` | **new** | Engine dependency graph (Cody CTO Mermaid, §engine:strategy) |
| `scenario-engine-dataflow.pdf` | **new** | API + scenario engine 16-node dataflow (Developer spec fed71685) |

## Data sources

- **Price and buy data**: `~/omnirisk/bot/logs/decisions.jsonl` — entries with `"ts": "2026-05-06T0..."` and `"action": "buy_executed"` (tier completions) or any `priceUsd`-carrying entry within the cycle window 01:00–06:30 UTC.
- **Liquidity data**: hardcoded point readings from `bot/SOL.md` (01:00 ≈ $5,418; 06:13 ≈ $6,098.2); the log does not record per-tick pool liquidity.
- **Ratchet step timestamps**: extracted from `decisions.jsonl` `"action": "alert"` entries containing `"ratchet step"`. Step 1 at `2026-05-06T02:19:13Z`, step 2 at `2026-05-06T05:51:51Z`.
- **Architecture + dependency graph**: static — from manuscript §engine:strategy and Cody CTO's Mermaid diagram (issue OMN-160).
- **Scenario dataflow nodes/edges**: static — from Developer sub-issue spec `fed71685` (issue OMN-160).

## Regeneration

```bash
cd papers/2026/omnirisk-rayachain-bittensor
python3 figures/_make_figures_polished.py
```

Requires: `matplotlib ≥ 3.9`, `networkx` (any recent), Python ≥ 3.9.
`pip3 install networkx` if missing.

## Dimensions

All figures are produced at the following `figsize` (inches, rendered as PDF at 72 dpi reference):
- Architecture, dependency graph: 10 × 6.5 and 9 × 7 respectively
- Price, liquidity charts: 10 × 5
- Scenario dataflow: 14 × 9

LaTeX includes these at default `\linewidth` scaling — no rasterisation.

## Baseline script

The original unstyled baseline figures were generated by `_make_figures.py` (committed with `e0ddd5e`). That script remains in place for reproducibility of the raw data layer. The polished script (`_make_figures_polished.py`, committed with OMN-160) replaces the three PDFs in-place and adds two new ones.
