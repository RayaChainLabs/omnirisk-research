# RECIPE — five-engine-dependency-graph (OMN-169 D2 rebuild)

## What the diagram says

Five engines compose over a Shared Fabric substrate; Bittensor provides killable verifiable scoring to the Prediction Engine; all three signal engines funnel forecasts/programs/signals into the Agentic Engine; the Agentic Engine routes actions through the Governance Gate (paused-by-default, board sign-off required).

## Data behind it

Static architecture — from long-paper §engine:strategy and the issue scaffold in OMN-169. No fabricated numbers; all labels match the paper's section language.

## Tool

Route B — D2 with ELK layout (`d2 --layout=elk`). Chosen because the 1-substrate + 5-engines + 1-gate topology benefits from ELK's layered edge routing, which eliminates crossings that Mermaid's auto-layout would produce.

## Source files

- `five-engine-dependency-graph.d2` — human-edited source of truth (OMN-169)
- `five-engine-dependency-graph.svg` — web embed
- `five-engine-dependency-graph.pdf` — vector for paper inclusion (LaTeX `\includegraphics`)
- `five-engine-dependency-graph.png` — 1602×2160 high-DPI raster

Both papers share these files: `papers/2026/agentic-iov-risk-focused/figures/` is a copy of the above.

## Render command

```bash
brew install d2   # one-time; version 0.7.1 used
cd papers/2026/omnirisk-rayachain-bittensor/figures

d2 five-engine-dependency-graph.d2 five-engine-dependency-graph.svg
d2 --layout=elk five-engine-dependency-graph.d2 five-engine-dependency-graph.pdf
d2 --layout=elk --scale=0.9 --pad=40 five-engine-dependency-graph.d2 five-engine-dependency-graph.png

# sync to focused paper
cp five-engine-dependency-graph.{d2,svg,pdf,png} \
  ../../agentic-iov-risk-focused/figures/
```

## Known limitations

- D2 places the Bittensor node inside the engines container flush-left; ELK positions it before Prediction Engine, creating a slightly longer dashed edge path — acceptable for readability.
- PDF produced via Chromium/Playwright headless (D2 0.7.1 default path); fonts are embedded via base64 data URIs in the SVG — safe for arXiv upload.
- Light background only; no dark-mode variant (acceptable for academic PDF; web embed uses the SVG directly).
