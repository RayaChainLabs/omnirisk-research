## Bundle contents (`submission/arxiv-bundle.tar.gz`)

The tarball is produced by `make arxiv` from
`papers/2026/agentic-iov-risk-focused/`. The recipe stages a clean copy,
strips macOS metadata (`.DS_Store`, `._*`), and excludes every build
artefact (`*.aux`, `*.bcf`, `*.blg`, `*.fdb_latexmk`, `*.fls`, `*.log`,
`*.out`, `*.run.xml`, `*.toc`, `*.dvi`, `*.ps`, `manuscript.pdf`,
the entire `build/` tree, and the local `references.bib` symlink).

| Path (relative to bundle root) | Source |
|---|---|
| `manuscript.tex` | `papers/2026/agentic-iov-risk-focused/manuscript.tex` |
| `manuscript.bbl` | generated by `make pdf` from the local `references.bib` symlink; ships in lieu of the `.bib` per arXiv guideline |
| `figures/omnirisk-architecture.png` | committed at `papers/2026/agentic-iov-risk-focused/figures/omnirisk-architecture.png` |
| `figures/five-engine-dependency-graph.pdf` | copied from the long paper (`papers/2026/omnirisk-rayachain-bittensor/figures/`) into the focused paper's `figures/` to make the path local |

**Why ship the `.bbl` and not the `.bib`.** The local `references.bib`
is a symlink (`papers/2026/agentic-iov-risk-focused/references.bib ->
../../../_bib/references.bib`) into the shared bibliography. arXiv
autotex's `tar` extraction may or may not preserve the symlink target
depending on the unpack step; shipping the `.bbl` removes the question
entirely. arXiv explicitly states: *"if you upload a `.bbl` file, its
filename must match the name of the main `.tex` file"* (which is the
case here: both are named `manuscript`). The `.bbl` header declares
`bbl format version 3.3`, matching TeX Live 2025+ (current arXiv default).

## Build instructions for arXiv

arXiv autotex runs `latexmk -pdf` (or equivalent) on the extracted
tarball. The manuscript builds cleanly against any TeX Live 2024+
install. The `seqsplit` package is referenced through a `¥providecommand`
fallback so the build succeeds whether the CTAN package is installed
or not (arXiv's recent TeX Live ships it). Bibliography style is
`numeric-comp` (renders IEEE-compatible numbered citations); the
camera-ready cut for IEEE TIFS will switch to `style=ieee` if
`biblatex-ieee` is available on the publisher's TeX.

## Tarball verification

`make arxiv-verify` extracts the bundle into a fresh tmpdir and runs
`latexmk -pdf -interaction=nonstopmode manuscript.tex` from a clean
working directory. The verified-clean signal is:

```
==> arxiv-verify OK: 15 pages
    fresh tmpdir: /var/folders/.../arxiv-verify.XXXXXX
```

A run with any `Warning`, `Underfull`, `Overfull`, `Missing`, or
`Undefined` line in the final pdflatex pass should fail before
submission.

## Forbidden patterns audit

The arXiv submission guidelines disallow several TeX patterns. The
manuscript was audited against the following list and is clean:

| Pattern | Reason | Status |
|---|---|---|
| `¥today` | arXiv rebuilds PDFs over time; the date will drift | clean |
| `¥pdfoutput=...` | branching breaks autotex routing | clean |
| `¥xr{...}` | external-document refs do not resolve on arXiv | clean |
| Embedded JavaScript / animation / movie objects | security restriction | clean |
| Unescaped `#` / `%` in URL macros | TeX special-character collision | clean |
| Cross-paper relative paths (`../...`) | sibling tree absent in extracted bundle | clean (was present in v1.1; resolved in v1.2 by copying the dependency-graph PDF into the focused paper's `figures/`) |

Re-run the audit at any time with:

